# ⚽ Football Arena · 足球竞技场

一个用**纯静态 HTML + Canvas** 实现的足球模拟小游戏，包含 **2026 美加墨世界杯** 淘汰赛对阵图 + **2025/26 欧冠**（联赛阶段 36 队 + 淘汰赛对阵图）+ **5 大联赛**（英超 / 西甲 / 意甲 / 德甲 / 法甲）近 5 个赛季的积分榜对战。无需任何构建工具或依赖，浏览器打开即可游玩。

## ✨ 功能特性

- 🏆 **世界杯淘汰赛对阵树**（`cups/worldcup.html`）：左右两侧上/下半区对阵图，点任意一场比赛开战。
- ⭐ **欧冠淘汰赛对阵图 + 联赛阶段榜**（`cups/ucl.html`）：2025/26 赛季 16 强起淘汰赛对阵（中央冠军柱显示巴黎卫冕），下方瑞士制 36 队联赛阶段积分榜，点击任意两支球队自定义对战。
- ⚽ **5 大联赛积分榜对战**（`leagues/league.html?c=england|spain|italy|germany|france`）：五大联赛共用一个页面模板，通过 URL 参数 `c=` 切换联赛，内置近 5 个赛季（2021/22 ~ 2025/26）积分榜，点击两支球队自定义主客队对战。
- 🎲 **随机抽队**：不想自己挑？点一下按钮 / 按空格键随机抽两队对战。
- 🎮 **canvas 球场物理模拟**：实时模拟球场与足球运动，进球判定严格——**整球需穿过门框且不碰立柱**才算进球。
- ⏱️ **完整比赛时钟**：常规时间 90 分钟，平局可配置进入加时赛至 120 分钟，再平局进入点球大战。
- 🚩 **球队标识**：世界杯页用 [flagcdn](https://flagcdn.com/) 国旗；联赛页用球队主客场色 + 3 字母缩写的动态队徽。
- ⚖️ **球队实力差异**：联赛 / 欧冠页按积分差让强队球目标速率更高（更频繁撞墙、更多进球机会）并叠加极轻的转向辅助（仅强队有，几乎看不出“找门”），强队胜率高于 50% 同时保留爆冷；世界杯页无数据，自动退回 50/50。

## 🚀 快速开始

直接用浏览器双击打开任意一个 HTML 即可，无需安装、无需构建：

- `cups/worldcup.html` — 世界杯淘汰赛
- `cups/ucl.html` — 2025/26 欧冠（联赛阶段 + 淘汰赛）
- `leagues/league.html?c=spain`（或 `england` / `italy` / `germany` / `france`）— 五大联赛，缺省参数时默认西甲

```bash
# 任选其一（打开世界杯页）
start cups/worldcup.html        # Windows
open cups/worldcup.html         # macOS
xdg-open cups/worldcup.html     # Linux
```

> 联赛页靠 `?c=` 参数区分联赛。双击打开时地址栏可直接补参数，或从任意页面左侧导航栏点进对应联赛。

## 🎯 玩法说明

1. 点击左右两侧对阵树中的任意一场比赛选中两队，或点「随机抽两队」按钮；
2. 点击 **「开始模拟」** 按钮开战；
3. 观看实时模拟，进球得分，90 分钟战平则按配置进入加时 / 点球；
4. 进球需整球穿过门框、不碰立柱。

## ⚙️ 配置项

比赛流程/物理/点球所有配置集中在 `js/engine.js` 顶部的 `// ---------- 配置 / 常量 ----------` 区域，用文本编辑器直接改即可，改完刷新浏览器生效。所有页面共享同一份引擎，改一次全生效。

### 比赛流程

| 配置项 | 默认值 | 说明 |
|---|---|---|
| `ALLOW_EXTRA` | `true` | 是否允许加时赛。`false` = 小组赛模式：90' 战平直接终场（平局），不进加时。由页面经 `initEngine(TEAM, { allowExtra })` 传入。 |
| `ALLOW_PENS` | `true` | 是否允许点球大战。`false` = 小组赛模式：加时结束仍平局直接终场，不进点球。仅在进入加时后才生效。由页面经 `initEngine(TEAM, { allowPens })` 传入。 |
| `T_REG` | `60` | 常规时间（0~90'）映射的真实秒数。改小 → 比赛更紧凑，改大 → 更慢。 |
| `T_EXTRA` | `20` | 加时赛（90~120'）映射的真实秒数（与常规同比例 0.667s/min）。 |
| `P_ACCEL` | `1` | 计时曲线指数。`1` = 匀速线性；`>1` = 前慢后快（缓入，临近终场节奏加快）。 |
| `KICKOFF_PAUSE` | `1.0` | 中圈开球后停顿秒数（开场 / 半场 / 进球后 / 加时开球都会触发）。 |

> **调试开关**：`js/engine.js` 顶部还有两个硬编码常量 `FORCE_EXTRA` / `FORCE_PENS`（均默认 `false`），它们不经 `opts` 暴露，需要直接编辑源码改值。`FORCE_PENS = true` 会让任何比分结束直接进点球，`FORCE_EXTRA = true` 会让 90' 结束强制进加时。

#### 流程优先级

当 90' 常规时间结束时，按以下顺序判断（前者命中则不再往下）：

1. `FORCE_PENS`（源码常量）→ 直接进点球大战
2. `FORCE_EXTRA`（源码常量）→ 强制进入加时
3. 平局且 `ALLOW_EXTRA` → 进入加时
4. 否则 → 终场（分出胜负，或小组赛平局收场）

加时 120' 结束时：`FORCE_PENS` 或（平局且 `ALLOW_PENS`）→ 点球大战；否则终场。

#### 常用预设

- **小组赛模式**：页面传入 `allowExtra: false` + `allowPens: false`（90' 战平即终场，记平局）
- **淘汰赛模式**（默认）：`allowExtra: true` + `allowPens: true`
- **调试点球大战**：改源码 `FORCE_PENS = true`（一开局就能看到点球）
- **调试加时赛**：改源码 `FORCE_EXTRA = true`

### 物理

| 配置项 | 默认值 | 说明 |
|---|---|---|
| `TARGET_SP` | `480` | 足球恒定目标速率（像素/秒）。等价于 `BASE`，向后兼容保留。调大球更快、比赛更激烈。 |

### 球队实力差异（速度差异 + 轻量转向）

强队打弱队时，强队获胜概率高于 50%，同时保留爆冷可能。机制以**速度差异为主、转向为辅**：

1. **速度差异**：每颗球有自己的目标速率 `b.sp`，强队球更快、弱队球更慢。强队球单位时间飞过更长路程、更频繁接触外圈（进球判定区），进球机会更多。
2. **轻量转向**：仅强队球每帧把速度方向朝球门中心轻掰一点（增益 `STEER_MAX` 很小，约为纯转向方案的 1/7，几乎看不出“找门”），作为辅助拉一点强队进球率。

只改速率 + 轻量方向、不改进球判定。势均力敌时两队速率都 = `BASE`、增益都 = 0，与原引擎完全一致。速率关于 `BASE` 对称，平均节奏不变。

引擎经 `TEAM.rating(t)` 读取球队实力：联赛页 = 当前赛季积分 `pts`（+ 净胜球 `gf-ga` ×0.1 微调，赛季切换时重建；所选赛季未开赛时自动回退到最近有数据的赛季）；欧冠页 = 联赛阶段积分（同微调）；**世界杯页无实力数据，不提供 `rating`，自动退回 50/50**。两队 `rating` 不存在或相等时同样退回 50/50。

| 配置项 | 默认值 | 说明 |
|---|---|---|
| `BASE` | `480` | 基础速率（= `TARGET_SP`）。势均力敌时两队都用此速率。 |
| `RATING_D` | `40` | Elo 分母。两队 rating 差经 `sA = 1/(1+10^(-(Ra-Rb)/D))` 映射成强队进球份额。越大越接近 50/50，越小差距拉得越开（主手感旋钮）。 |
| `UPSET` | `0.30` | 冷门旋钮（0~1）。`sA' = lerp(sA, 0.5, UPSET)`，把强队份额往 0.5 拉，保留爆冷空间。 |
| `SPREAD` | `192` | 最大速率差幅度（±40%）。强队最快 `BASE+SPREAD`、弱队最慢 `BASE-SPREAD`。调大强弱更悬殊但两球快慢差异更明显。 |
| `STEER_MAX` | `0.15` | 强队最大转向增益（rad/s，仅辅助）。很小，几乎看不出“找门”；调大强队更强势但“变向”更明显。仅强队有，弱队无。 |
| `PEN_RATING_AMP` | `0.15` | 点球命中率按实力份额浮动幅度（次要，优先级低于常规时间速率差）。 |

> **效果-观感权衡**：本方案在当前引擎（每场进球率 < 1 球）下纯速度差异效果偏弱，故叠加轻量转向辅助。模拟多局（以联赛积分为 rating）约得：均势 ~50/50；中差强队 ~43%；大差强队 ~52%、弱队仍 ~21% 爆冷；极大差强队 ~53%、弱队 ~20% 爆冷。转向增益仅 0.10 rad/s 量级，球几乎不显“找门”。常量集中在 `js/engine.js` 顶部 `TARGET_SP` 附近的「球队实力差异」配置区。

### 点球大战

| 配置项 | 默认值 | 说明 |
|---|---|---|
| `PEN_GOAL_P` | `0.72` | 单粒点球进球概率（0~1）。调高更易进、调低更易扑。 |
| `PEN_GC` | `-Math.PI/2` | 点球阶段球门固定角度（球场正上方），一般无需改。 |
| `PEN_PREP` | `0.5` | 助跑准备阶段秒数。 |
| `PEN_FLIGHT` | `0.85` | 球飞行阶段秒数。 |
| `PEN_RESOLVE` | `0.9` | 进球/扑救落定阶段秒数。 |
| `PEN_GAP` | `0.6` | 两轮点球之间的间隔秒数。 |

## 🛠️ 技术栈

- 纯原生 **HTML / CSS / JavaScript**，零依赖，零构建
- **Canvas 2D** 绘制球场与足球动画
- CSS/JS 按功能模块拆分（`base.css` 通用 / `standings.css` 积分榜 / `bracket.css` 对阵图；`engine.js` 引擎 / `standings.js` 联赛 / `bracket.js` 世界杯 / `ucl.js` 欧冠），通过 TEAM 适配器复用引擎
- 复用优先的结构：五大联赛共用 `leagues/league.html` + `js/league-page.js`（`?c=` 参数化）；世界杯与欧冠对阵图渲染抽到 `js/bracket-render.js` 公共模块；新赛季空表由 `js/season-util.js` 生成，避免手抄全 0 数据
- 世界杯页国旗资源来自 [flagcdn](https://flagcdn.com/)（支持 CORS）

### 想接入 HTTP 数据加载

目前所有球队/赛季数据都写死在各 HTML 内联 `<script>` 里，因为 `file://` 协议下浏览器禁止 `fetch()` 读本地 JSON。若日后要把数据外置到 `data/*.json`，需通过 HTTP 服务器访问：

```bash
python -m http.server 8000
# 打开 http://localhost:8000/leagues/league.html?c=spain
```
