# 球队实力差异设计方案（转向偏置）

> 目标：让强队打弱队时，强队获胜概率明显高于 50%，同时保留爆冷可能，并且视觉上自然、不违和。
> 本文档只描述方案，暂不含代码实现。

## 1. 背景与问题

当前 `js/engine.js` 的比赛引擎里，两颗球在圆形场地内以**恒定速率**乱弹，谁的球穿过旋转的球门弧就得分。整个过程对双方完全对称，因此无论强弱，两队进球期望都是 50/50。用户希望引入球队实力差异，让强弱对战更接近真实赛事。

曾考虑过的**进球闸门方案**（球过线后再用概率判定是否算进球）已被否决：球都过门线了还要“骰子”决定，用户和观众都难以接受，视觉上很假。

**已确定采用转向偏置（steering bias）方案。**

## 2. 核心思路

不改变“速率恒定”和“球过线=进球”这两条铁律，只在每帧积分时，**把球的速度方向朝当前球门中心轻轻掰一点点**，掰的力度由球队实力评分（rating）决定：

- 强队的球更“有目的性”，更频繁地朝门口逼近，进球更多。
- 弱队的球基本还是乱弹，但仍然会自然地偶尔进球。
- 因为只旋转速度方向、随后仍由现有归一化保持速率恒定，所以两队球速看起来一样，进球依旧靠真实穿过门线，观感自然。

### 相关引擎位置（现状）

- `integrate(h)`（`js/engine.js` 约 285 行）：`b.x += b.vx*h` 后把速度重新归一化到 `TARGET_SP=480`。**转向偏置就加在这里**——归一化之前先旋转方向，归一化会自动保持速率。
- 球门中心角度：`state.goalCenter + state.rot`（球门在旋转，见 `wallAndGoal` 约 296 行 `const gc = state.goalCenter + state.rot`）。
- 计分：`wallAndGoal()` 中球进门口且 `dist >= R` 时调 `scoreGoal(b)`（约 330 行）。这条逻辑**完全不动**。
- 两颗球：`state.balls[0]` 为主队（tA，红），`state.balls[1]` 为客队（tB，蓝），`b.ref` 指向球队对象。

## 3. 转向偏置的实现要点

每帧对每颗球：

1. 计算球门中心当前角度 `gc = state.goalCenter + state.rot`，以及从球指向门口中心的方向角 `target = atan2(gcY - b.y, gcX - b.x)`（门口中心点在半径 `R` 上）。
2. 计算当前速度方向角 `cur = atan2(b.vy, b.vx)`。
3. 求最小夹角差 `d = normAngle(target - cur)`。
4. 按增益把方向朝目标掰一点：`newAng = cur + clamp(d, -maxTurn, maxTurn)`，其中 `maxTurn = gain * h`（`gain` 由该球所属球队的实力份额决定）。
5. 用 `newAng` 重建 `vx, vy`（模长任意），随后交给现有归一化恢复到 `TARGET_SP`。

关键点：**只改方向、不改速率**，所以并排看两颗球速度一致，唯一差异是强队球“更爱往门口跑”。

## 4. 实力评分（rating）来源

引擎通过 `TEAM.rating(t)` 读取球队实力；**若该函数不存在或两队 rating 相等，则退回当前的 50/50 行为**（世界杯页即如此）。各页面适配器分别提供：

- **联赛页**（`js/badge.js` 的 `makeBadgeTeamAdapter` + `js/standings.js`）：用最近赛季积分 `pts`（可掺净胜球 `gf-ga` 作微调）。数据在页面内联的全局 `SEASONS[0].clubs`（结构 `{code,pld,w,d,l,gf,ga,pts}`），`CLUBS[code]` 是队对象。适配器在 `badge.js` 生成，而 `SEASONS` 在页面/standings 作用域，因此让 `makeBadgeTeamAdapter` 接收一个可选的 `rating` 回调，或由 standings 覆盖 `TEAM.rating`。
- **欧冠页**（`js/ucl.js`）：用联赛阶段排名数组 `LEAGUE`（`{code,pts,...}`）里的积分。
- **世界杯页**（`js/bracket.js`，国旗版 TEAM）：无实力数据，`rating` 返回常量 → 自动退回 50/50。

## 5. 概率与手感设计

把两队 rating 差映射成强队的“进球份额” `sA`，再映射成转向增益差：

- **Elo logistic 映射**：`sA = 1 / (1 + 10^(-(Ra - Rb)/D))`。`D` 是主手感旋钮，越大越接近 50/50，越小越拉开差距。
- **冷门旋钮 `UPSET`（0~1）**：`sA' = lerp(sA, 0.5, UPSET)`，把份额往 0.5 拉，保留爆冷空间。
- **弱队保底增益**：即使弱队份额低，也给一个最小转向增益，避免弱队 0 进球。
- 由 `sA'` 分别得到两队的 `gain`（强队高、弱队低），差距映射到转向力度。

### 默认取值（保守起步）

先把参数调保守：强队赢面明显、但**不到单机碾压**，先出效果再逐步微调 `D` / `UPSET` / `gain` 上限。这些常量集中放在配置区（`TARGET_SP` 附近）便于调参。

## 6. 点球大战（次要）

点球命中率 `PEN_GOAL_P = 0.72` 目前两队相等。可作为可选项按实力微调（强队命中率略高），优先级低于常规时间的转向偏置。

## 7. 备选方案（未采用，仅备忘）

只给强队略高速率（如 500 vs 460）——改动仅一行，但并排看两球速度明显不同，耐看度不如转向偏置，因此不采用。

## 8. 落地步骤

1. 在 `engine.js` 的 `integrate`（或其后）加入转向偏置逻辑，并在配置区新增 `D` / `UPSET` / `gain` 相关常量。
2. 给三处 TEAM 适配器补 `rating(t)`：`badge.js`/`standings.js`、`ucl.js`、`bracket.js`。
3. 可选：`PEN_GOAL_P` 按实力微调。
4. 浏览器验证三个页面（`leagues/league.html?c=spain`、`cups/ucl.html`、`cups/worldcup.html`）：跑多局确认强队胜率明显 > 50% 且弱队仍偶胜、无报错。
